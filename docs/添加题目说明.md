# 添加题目完整说明

本文档适用于项目 **nani-kiru**（日麻切牌练习）。

数据源位置：`public/data/`。题目存放在 `public/data/categories/<分类id>/problems/` 下。

---

## 一、前置：确认有分类

若还没有分类，先添加分类：

```bash
npm run add:category -- --id <分类id> --title "<分类显示名>"
```

示例：

```bash
npm run add:category -- --id efficiency --title "门清效率"
```

查看已有分类：

```bash
npm run add:problem -- --list-categories
```

---

## 二、用脚本添加题目（推荐）

**除 `--id` 外，其余参数均为必填**；`--id` 不传则自动生成。

在项目根目录下执行：

```bash
npm run add:problem -- \
  --category <分类id或标题> \
  --title "<题目名>" \
  --round "<局数>" \
  --seat <自风> \
  --turn <巡目> \
  --dora "<宝牌>" \
  --hand "<14张手牌>" \
  --answer "<切牌>" \
  --shanten <向听> \
  --eff "<牌:数值,...>" \
  --reasoning "<解析>"
```

### 必填参数一览

| 参数 | 说明 | 校验规则 |
|------|------|----------|
| `--category` | 分类 id 或分类标题 | 须在 `categories.json` 中存在 |
| `--title` | 题目标题 | 非空字符串 |
| `--round` | 局数 | **东南西北** + **数字** + 可选**局**，如 `东1局`、`南2`、`西3局`；缺「局」会由程序自动补上 |
| `--seat` | 自风 | 仅允许 `east` / `south` / `west` / `north` |
| `--turn` | 巡目 | 非负整数 |
| `--dora` | 宝牌 | 至少 1 张，每张为合法牌码 |
| `--hand` | 手牌 | **必须恰好 14 张**，每张为合法牌码 |
| `--answer` | 正确答案（切哪张） | 1 张合法牌码 |
| `--shanten` | 向听数 | 整数 |
| `--eff` | 牌效率 | 至少一组 `牌:数值`，如 `6s:18,8s:14,white:10`；牌码合法、数值为数字 |
| `--reasoning` | 解析文案 | 非空；其中的 `[牌码]` 须为合法牌码。需要换行时在字符串里写 `\n`，如 `"第一行\n第二行"` |

### 可选参数

| 参数 | 说明 |
|------|------|
| `--id` | 题目唯一 id；不传则自动生成 `<分类id>-<时间戳>` |

### 牌码规则（合法字符串）

- 万子：`1m`–`9m`
- 筒子：`1p`–`9p`
- 索子：`1s`–`9s`
- 字牌：`east` / `south` / `west` / `north` / `white` / `green` / `red`
- 背面：`back`

解析中的牌用 `[牌码]` 包裹会渲染为牌图，如 `切[6s]`。**换行**：在 `--reasoning` 里写 `\n` 会存成真实换行，例如 `"第一行\n第二行"`。

### 完整示例

```bash
npm run add:problem -- \
  --category efficiency \
  --title "8巡速度优先" \
  --round "东1局" \
  --seat south \
  --turn 8 \
  --dora "7p" \
  --hand "2m 3m 4m 4m 5m 6m 2p 3p 4p 6s 7s 8s white white" \
  --answer "6s" \
  --shanten 1 \
  --eff "6s:18,8s:14,white:10" \
  --reasoning "当前手牌以中张两面改良为主，切[6s]可最大化受入并保留更好的进张结构。"
```

**局数可省略「局」**，程序会自动补全：

```bash
--round "南2"    # 存储为 南2局
```

**指定题目 id：**

```bash
npm run add:problem -- ... --id shape-001
```

脚本会：

1. 在 `public/data/categories/<分类id>/problems/` 下生成 `<题目id>.json`
2. 自动更新该分类下的 `titles.json`（题目列表映射）

校验不通过时会在终端列出错误并退出，不写入文件。

## 三、手动添加题目

1. 在 `public/data/categories/<分类id>/problems/` 下新建 `<题目id>.json`。
2. 内容格式参考同目录下已有题目，例如：

```json
{
  "id": "题目唯一id",
  "category": "分类显示名",
  "category_id": "分类id",
  "category_title": "分类显示名",
  "title": "题目标题",
  "round_label": "东1局",
  "seat_wind": "south",
  "turn": 8,
  "dora": ["7p"],
  "hand_tiles": ["2m", "3m", "4m", "4m", "5m", "6m", "2p", "3p", "4p", "6s", "7s", "8s", "white", "white"],
  "answer_discard": "6s",
  "shanten": 1,
  "tile_efficiency": { "6s": 18, "8s": 14, "white": 10 },
  "reasoning": "解析文案，牌用[6s]会渲染为牌图。"
}
```

3. 手动编辑该分类下的 `public/data/categories/<分类id>/titles.json` 增加对应条目，或执行 `npm run build:problems` 由脚本根据 `problems/` 重新生成 `titles.json` 与 `problems.index.json`。

---

## 四、添加后校验与构建

```bash
npm run validate:problems   # 校验题目 JSON 与分类配置
npm run build:problems     # 刷新 problems.index.json 与各分类 titles.json（不删已有题目文件）
```

---

## 五、流程小结

| 步骤 | 指令 |
|------|------|
| 1. 确保有分类 | `npm run add:category -- --id <id> --title "<显示名>"` 或 `npm run add:problem -- --list-categories` |
| 2. 添加题目 | `npm run add:problem -- --category ... --title ... --round ... --seat ... --turn ... --dora ... --hand ... --answer ... --shanten ... --eff ... --reasoning ...`（仅 `--id` 可选） |
| 3. 校验 | `npm run validate:problems` |
| 4. 刷新索引（可选） | `npm run build:problems` |

数据与脚本均在项目根目录下，以上命令均在项目根目录执行。
